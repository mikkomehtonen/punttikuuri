# Prefill Set Values from Previous Set

## Context

When logging multiple sets of the same exercise, users typically perform sets with similar or identical weight and repetitions. Currently, the weight and reps inputs start empty for every new set, requiring the user to re-enter the same values repeatedly. This slows down workout logging during training sessions.

This feature pre-fills the weight and reps inputs with the values from the most recently logged set of the same exercise, allowing the user to quickly confirm or adjust the values before logging the next set.

## Out of Scope

- Editing or deleting previously logged sets.
- Prefilling from a different exercise's sets.
- Remembering values across browser sessions or devices beyond what is already stored in the database.
- Preserving user-typed values across form validation errors (the form will re-prefill from the last logged set on reload).

## Implementation approach

The server-side `load` function in `src/routes/exercises/[id]/+page.server.ts` already fetches `todaySets` (ordered by `set_number` ascending) and `previousSessions` (ordered by `workout_date` descending, sets within each session ordered by `set_number` ascending). A new `lastSet` field will be derived from this existing data and added to the returned page data:

1. If `todaySets` is non-empty, `lastSet` is the last element of `todaySets` (highest `set_number`).
2. Else if `previousSessions` is non-empty and its first session has sets, `lastSet` is the last set of that first (most recent) session.
3. Else `lastSet` is `null`.

The client-side `+page.svelte` will initialize the `weight` and `reps` `$state` variables from `data.lastSet` when it is non-null, converting the numeric values to strings. When `lastSet` is `null`, the inputs remain empty (current behavior).

Weight formatting: use JavaScript's default `String()` conversion, which produces `"80"` for `80.0` and `"72.5"` for `72.5` — matching the existing display format in the sets list.

## Tasks

### Task 1 - Server: Add lastSet to exercise detail page data

- todaySets has sets (e.g., `[{set_number:1, weight_kg:80, repetitions:10}, {set_number:2, weight_kg:100, repetitions:5}]`) + page loads
  - → `lastSet` is `{weight_kg: 100, repetitions: 5}` (last element of todaySets)
- todaySets is empty and previousSessions has sessions with sets (e.g., first session sets: `[{set_number:1, weight_kg:60, repetitions:12}, {set_number:2, weight_kg:60, repetitions:10}]`) + page loads
  - → `lastSet` is `{weight_kg: 60, repetitions: 10}` (last set of most recent session)
- todaySets is empty and previousSessions is empty + page loads
  - → `lastSet` is `null`
- todaySets is empty and previousSessions has a session with no sets (empty sets array) + page loads
  - → `lastSet` is `null`

### Task 2 - Client: Prefill form inputs from lastSet

- lastSet is `{weight_kg: 100, repetitions: 5}` + page renders
  - → weight input value is `"100"`
  - → reps input value is `"5"`
- lastSet is `{weight_kg: 72.5, repetitions: 8}` + page renders
  - → weight input value is `"72.5"`
  - → reps input value is `"8"`
- lastSet is `null` + page renders
  - → weight input value is `""` (empty)
  - → reps input value is `""` (empty)

## Technical Context

- SvelteKit 2.63.1 with Svelte 5.56.3 — no new dependencies required.
- Existing server-side rendering tests use `render()` from `svelte/server` with a `makeData()` helper; new tests follow the same pattern.
- The `lastSet` type is `{weight_kg: number; repetitions: number} | null`.
- The `PageData` type in `./$types` is auto-generated by SvelteKit from the `load` function return type; no manual type updates needed beyond the `load` return value.
